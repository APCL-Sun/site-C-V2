# APCL 연구실 홈페이지 수정 방법

이 문서는 **연구실 홈페이지 내용을 업데이트하는 후배님들을 위한 안내서**입니다.
코딩을 몰라도 아래 순서대로만 따라하시면 됩니다.

여러분이 수정할 파일은 **딱 3개**뿐입니다.

| 파일 | 내용 |
|---|---|
| `_data/members.yml` | 교수님/대학원생 명단, 사진 |
| `_data/publications.yml` | 국내논문 / 국외논문 / 특허 / 학술대회 |
| `_data/news.yml` | News & Gallery 소식 |

이 3개 파일 외에는 (레이아웃, CSS, 다른 페이지 등) **건드리지 않아도** 홈페이지가 정상적으로 나옵니다.
잘 모르겠으면 다른 파일은 절대 수정하지 마세요.

Publications와 News 페이지는 이제 **검색창과 페이지 넘기기(페이지네이션)**가 자동으로 붙어 있어서,
`_data/` 파일에 항목을 추가하기만 하면 화면에서 자동으로 검색되고 페이지가 나뉩니다. 따로 설정할 것은 없습니다.

---

## 0. 수정하는 방법 (2가지 중 편한 것 선택)

### 방법 A. GitHub 웹사이트에서 바로 수정 (추천, git 몰라도 가능)

1. GitHub 저장소 페이지에 접속합니다.
2. 고치고 싶은 파일(`news.md` 등)을 클릭합니다.
3. 화면 오른쪽 위 **연필 아이콘(Edit this file)**을 클릭합니다.
4. 아래 안내에 맞춰 내용을 추가/수정합니다.
5. 화면 아래로 내려서 **"Commit changes..."** 버튼을 누릅니다.
   - 커밋 메시지는 대충 `news 업데이트` 처럼 무엇을 했는지만 적으면 됩니다.
   - "Commit directly to the main branch" 선택되어 있는지 확인 후 **Commit changes** 클릭.
6. 1~2분 정도 기다리면 자동으로 홈페이지에 반영됩니다.
   - 저장소 상단 **Actions** 탭에서 초록색 체크(✓)가 뜨면 반영 완료, 빨간 X가 뜨면 실패한 것입니다.
   - 실패했다면 아래 "자주 하는 실수" 항목을 확인해주세요.

### 방법 B. 컴퓨터에 다운받아서(git) 수정

Git과 GitHub Desktop 같은 프로그램에 익숙하다면 평소처럼 `git pull` → 파일 수정 →
`git add` → `git commit` → `git push` 하시면 됩니다. 익숙하지 않다면 방법 A를 사용하세요.

> **사진을 새로 추가하는 경우**에는 방법 A로는 한 번에 여러 파일을 올리기 불편할 수 있습니다.
> 저장소 메인 화면에서 `assets/images/members/` 또는 `assets/images/news/` 폴더로 들어간 뒤
> **Add file → Upload files** 버튼으로 사진 파일을 끌어다 놓으면 됩니다.

## 1. `_data/members.yml` — 구성원 명단 수정

### 1-1. 사진 올리기
- 사진 파일을 준비합니다 (정사각형에 가까운 비율 추천, 예: `park-ms.jpg`).
- 파일명은 영문/숫자/하이픈(-)만 사용하세요. **띄어쓰기, 한글 파일명 금지.**
- GitHub 저장소의 `assets/images/members/` 폴더에 업로드합니다.

### 1-2. 신입생 추가하기
`_data/members.yml` 파일을 열고, 아래처럼 **석사과정(`ms_students`)** 또는
**박사과정(`phd_students`)** 목록 맨 아래에 블록을 하나 추가합니다.

```yaml
  - name_kr: "여기에 이름"
    name_en: "여기에 영문 이름"
    year: "석사 1차"
    email: "본인이메일@chungbuk.ac.kr"
    photo: "/assets/images/members/파일명.jpg"
```

⚠️ **주의사항 (매우 중요합니다)**
- 맨 앞 `  - name_kr:` 부분의 **띄어쓰기(들여쓰기) 칸 수를 위/아래 다른 항목과 똑같이** 맞춰야 합니다.
  들여쓰기가 하나라도 다르면 홈페이지 전체가 깨질 수 있습니다.
- `이름:` 뒤에는 **반드시 띄어쓰기 한 칸**이 있어야 합니다. (`name_kr:"홍길동"` ❌ / `name_kr: "홍길동"` ✅)
- 값은 큰따옴표(`" "`)로 감싸주세요.
- 확실치 않으면 바로 위/아래에 있는 기존 사람 항목을 **통째로 복사**한 뒤, 이름/이메일/사진 경로만 바꾸는 것이
  가장 안전합니다.

### 1-3. 졸업생 추가하기
`alumni:` 항목 아래에 아래 형식으로 추가합니다.

```yaml
  - name_kr: "이름"
    degree: "석사 졸업 (2026)"
    current: "취업한 회사/기관명 (선택 입력)"
```

## 2. `_data/publications.yml` — 논문/특허/학술대회 수정

4개 항목(`domestic`, `international`, `patents`, `conferences`) 중
추가하고 싶은 곳에 아래 형식을 **맨 위에** 추가합니다. (최신순으로 보이도록 항상 위에 추가)

**국내논문 / 국외논문 / 학술대회 (형식 동일)**
```yaml
  - year: 2026
    title: "논문 제목 그대로 입력"
    authors: "저자1, 저자2"
    venue: "학술지명, 권(호), 페이지  또는  학회명"
```

**특허**
```yaml
  - year: 2026
    title: "특허 제목"
    inventors: "발명자1, 발명자2"
    number: "출원/등록번호"
```

⚠️ members.yml과 마찬가지로 **들여쓰기**와 **콜론 뒤 띄어쓰기**를 꼭 지켜주세요.
가장 쉬운 방법은 바로 위에 있는 기존 항목을 복사해서 내용만 바꾸는 것입니다.

## 3. `_data/news.yml` — News & Gallery 소식 추가

파일을 열면 예시 소식들이 아래와 같은 형식으로 되어 있습니다. **맨 위에** 아래 블록을 통째로 복사해서
붙여넣고 날짜/제목/내용/사진만 바꿔주세요. (새 소식은 항상 맨 위 = 최신순 정렬)

```yaml
- date: "2026.08.20"
  title: "여기에 제목"
  content: >
    여기에 소식 내용을 자유롭게 작성하세요.
    여러 줄로 적어도 됩니다.
  images:
    - "/assets/images/news/파일명1.jpg"
    - "/assets/images/news/파일명2.jpg"
```

사진이 없는 소식이면 `images:` 를 빈 목록으로 두면 됩니다.

```yaml
  images: []
```

사진은 `_data/members.yml`과 마찬가지로 `assets/images/news/` 폴더에 먼저 업로드한 뒤,
파일명 부분만 실제 업로드한 이름으로 바꿔주면 됩니다.

⚠️ **주의사항**
- `date:`, `title:` 뒤에는 큰따옴표(`" "`)로 감싼 텍스트, **콜론 뒤 띄어쓰기 한 칸** 필수.
- `content: >` 다음 줄부터는 **반드시 위 예시와 같은 들여쓰기(4칸)**를 지켜서 내용을 적어주세요.
- 사진 목록의 각 줄은 `- "경로"` 형태로, 맨 앞에 하이픈(-)과 띄어쓰기가 있어야 합니다.
- 확실치 않으면 바로 아래 있는 기존 소식 블록을 통째로 복사해서 내용만 바꾸는 게 가장 안전합니다.

이렇게 등록한 소식은 실제 홈페이지에서 **목록으로 자동 표시**되고, 클릭하면 사진과 전체 내용을 보여주는
팝업이 뜨며, 소식이 많아지면 자동으로 페이지가 나뉘고 검색창에서 제목/내용으로 검색도 됩니다.
화면 쪽은 따로 손댈 필요가 없습니다.

## 4. 자주 하는 실수 (Actions에서 빨간 X가 뜰 때)

1. **들여쓰기 문제 (가장 흔함)** — `.yml` 파일에서 스페이스 칸 수가 위/아래 항목과 다름.
   → 기존 항목을 복사해서 쓰면 대부분 해결됩니다.
2. **콜론 뒤 띄어쓰기 누락** — `name_kr:"홍길동"` 처럼 콜론 바로 뒤에 띄어쓰기가 없는 경우.
3. **따옴표를 안 닫음** — `title: "논문 제목` 처럼 큰따옴표가 한쪽만 있는 경우.
4. **목록 항목 앞에 하이픈(-) 누락** — `images:` 나 각 인물/논문 항목은 앞에 `- `(하이픈+띄어쓰기)가 있어야 합니다.

수정 후 GitHub 저장소 상단의 **Actions** 탭을 클릭하면 빌드 로그를 볼 수 있고,
어느 줄에서 에러가 났는지 영어로 나옵니다 (`_data/members.yml` 몇 번째 줄 등).
잘 모르겠으면 캡처해서 랩장/관리자에게 문의해주세요.

---

## 5. 반영까지 걸리는 시간

- 커밋(저장) 후 보통 **1~3분 이내**에 실제 홈페이지(`https://apcl.chungbuk.ac.kr`)에 반영됩니다.
- 반영 안 된 것처럼 보이면, 브라우저 새로고침을 **강력 새로고침**(Ctrl+Shift+R 또는 Cmd+Shift+R)으로 해보세요.
  (브라우저가 예전 화면을 캐시에 저장해두는 경우가 많습니다.)
